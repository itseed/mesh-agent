# MeshAgent — Plan 5: Deployment

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy MeshAgent ขึ้น DigitalOcean Droplet ด้วย Docker Compose production config, Nginx reverse proxy, HTTPS ผ่าน Let's Encrypt, และ `.env` production setup

**Architecture:** Docker Compose production มี services ทั้งหมด (web, api, orchestrator, db, redis, nginx) Nginx ทำ reverse proxy ไป web (port 3000) และ api (port 3001) HTTPS ด้วย Certbot Let's Encrypt Deploy ผ่าน `deploy.sh` script ที่ SSH เข้า Droplet แล้วรัน `docker compose pull && docker compose up -d`

**Tech Stack:** Docker Compose 2, Nginx 1.25, Certbot, Ubuntu 22.04, GitHub Actions (optional CI)

**Prerequisite:** Plans 1-4 เสร็จแล้ว, มี DigitalOcean Droplet (Ubuntu 22.04) และ domain name ชี้มาที่ Droplet IP

      AUTH_PASSWORD: ${AUTH_PASSWORD}

      GITHUB_WEBHOOK_SECRET: ${GITHUB_WEBHOOK_SECRET}

      redis:
        condition: service_healthy
      orchestrator:
        condition: service_started

    build:
      context: .
      dockerfile: apps/web/Dockerfile
      args:
        NEXT_PUBLIC_API_URL: https://${DOMAIN}/api
        NEXT_PUBLIC_WS_URL: wss://${DOMAIN}/api
    restart: unless-stopped
    environment:
      NODE_ENV: production
    depends_on:
      - api

  nginx:
    image: nginx:1.25-alpine
    restart: unless-stopped
    ports:
      - '80:80'
      - '443:443'

      - ./nginx/meshagent.conf:/etc/nginx/conf.d/default.conf:ro

## Task 3: Nginx Config

**Files:**

- Create: `nginx/meshagent.conf`

- [ ] **Step 1: Create `nginx/meshagent.conf`**

```nginx
# HTTP → HTTPS redirect + certbot challenge
server {
    listen 80;
    server_name ${DOMAIN};

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

# HTTPS
server {
    listen 443 ssl;
    server_name ${DOMAIN};

    ssl_certificate /etc/letsencrypt/live/${DOMAIN}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/${DOMAIN}/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;

    # Frontend (Next.js)
    location / {
        proxy_pass http://web:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # Backend API
    location /api/ {
        rewrite ^/api/(.*) /$1 break;
        proxy_pass http://api:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket
    location /api/ws {
        rewrite ^/api/(.*) /$1 break;
        proxy_pass http://api:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }
}
```

**หมายเหตุ:** Nginx config ใช้ `${DOMAIN}` เป็น placeholder — ก่อน deploy ให้แทนค่า domain จริง หรือใช้ `envsubst` ใน `deploy.sh`

## Task 5: Deploy Script

**Files:**

- Create: `scripts/deploy.sh`

- [ ] **Step 1: Create `scripts/deploy.sh`**

```bash
#!/usr/bin/env bash
# Deploy MeshAgent ไปยัง DigitalOcean Droplet
# Usage:
#   LOCAL: DROPLET_HOST=1.2.3.4 DROPLET_USER=root bash scripts/deploy.sh
#   ON SERVER: bash scripts/deploy.sh  (ถ้ารันบน Droplet ตรงๆ)

set -euo pipefail

DEPLOY_DIR="/opt/meshagent"
COMPOSE_FILE="docker-compose.prod.yml"

# ถ้ามี DROPLET_HOST = deploy จาก local ผ่าน SSH
if [[ -n "${DROPLET_HOST:-}" ]]; then
  USER="${DROPLET_USER:-root}"
  echo "→ Syncing files to ${USER}@${DROPLET_HOST}:${DEPLOY_DIR}..."
  rsync -az --exclude='.git' --exclude='node_modules' --exclude='.next' \
    . "${USER}@${DROPLET_HOST}:${DEPLOY_DIR}/"
  echo "→ Running deploy on server..."
  ssh "${USER}@${DROPLET_HOST}" "cd ${DEPLOY_DIR} && bash scripts/deploy.sh"
  exit 0
fi

# รันบน server
echo "→ Building and starting services..."
cd "$DEPLOY_DIR"

[[ -f .env ]] || { echo "Error: .env not found at ${DEPLOY_DIR}/.env"; exit 1; }

source .env

# Substitute DOMAIN in nginx config
envsubst '${DOMAIN}' < nginx/meshagent.conf > /tmp/meshagent.conf.rendered
cp /tmp/meshagent.conf.rendered nginx/meshagent.conf.rendered

docker compose -f "$COMPOSE_FILE" build --parallel
docker compose -f "$COMPOSE_FILE" up -d

echo ""
echo "✓ MeshAgent deployed. Running services:"
docker compose -f "$COMPOSE_FILE" ps
```

## Troubleshooting

**Services ไม่ start:**

```bash
docker compose -f docker-compose.prod.yml logs api
docker compose -f docker-compose.prod.yml logs orchestrator
```

**Database migration ไม่รัน:**

```bash
docker compose -f docker-compose.prod.yml exec api node -e "
  import('./dist/plugins/db.js').then(...)
"
# หรือ รัน migrate manually จาก packages/shared
```

**Claude Code ไม่มี auth:**

```bash
docker compose -f docker-compose.prod.yml exec orchestrator claude auth login
# จะได้ URL ให้เปิดใน browser แล้ว copy token กลับมา
```
